  1. What is reactive programming?

    What is reactive programming?

    • Reactive programming is a programming paradigm for the processing of streams of events.
    • Reactive programming can be seen as the the many-event equivalent of
      • event handlers
        similarity:
        events are handled as they come in
        difference:
        event handlers treat events individually in isolation, reactive programming treats them collectively
      • promises
        similarity:
        provide an explicit mechanism for asynchronous treatment of events
        difference:
        promises are associated with a single evernt, reactive programming with streams of events
    • Reactive programming can also be compared to functional programming on data sequences, except that with reactive programming the sequence is not defined or available in advance.
    • The processing of events in reactive programming is asynchronous i.e. the required processing is specified in advance and executed whenever the vents arrive.
    • Reactive programming uses the declarative style. The programmer specifies what needs to be done with the events or data associated with them, but the reactive programming library provides the functionality to do it.
    • Reactive programming is based on the observer pattern of object-oriented programming.

  2. Some use cases

    Some use cases

    • Reactive programming can be used with streams (JavaScript Streams API documentation can be found here), for the processing of video 'on the fly'.
    • It can be used to process the stream of events, where an event handler would process the events one by one.
