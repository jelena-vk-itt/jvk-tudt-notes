  1. What are frameworks and libraries?

    What are frameworks and libraries?

    • Conventionally, the difference between a framework and a library in software development is in the way that they are used.
      • Libraries are called by the code that uses them, which also drives execution and is in control:
        • the library provides some useful functionality, 'prepared earlier'
        • the code using the library is the application and it incorporates the library functionality to help it do something
      • Frameworks, on the other hand, constitute a complete application (if skeletal!) and are used through an 'inversion of control':
        • the framework is in the driving seat but devoid of content
        • the content is provided by the code that is utilising the framework
    • In the case of a client-side application, this distinction is somewhat blurred because of how these applications run - in the browser, which is effectively always in the driver seat.
      • Thus a client-side framework (e.g. Angular) has little opportunity to take true control:
        • it is 'sort of' in control at load time
        • generally gets its status as a framework through substantial functionality such as
          • routing
          • static site generation (SSG) and server-side rendering (SSR) builds
          • 'glue' artefacts that compose the application out of components
      • A library (e.g. React) typically provides a single development mechanism such as components

  2. Angular

    Angular

    • A client-side framework
    • Uses the Typescript language
      • developed by Microsoft
      • typed version of Javascript
    • Installed as an npm package
    • Development tools and constructs
      • components
      • templates with data binding
      • component 'glue' such as inputs, injection of services, directives
      • interfaces (data types)
      • routing
      • building to SSG and SSR
