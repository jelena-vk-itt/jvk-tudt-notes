  1. What is accessibility in Web applications?

    What is accessibility in Web applications?

    • An accessible Web site or application is one that allows anyone to use all its features, regardless of how they access the Web, especially people with physical or mental impairments .
    • Certain ways of developing in HTML, CSS and JavaScript are better than others and are deemed best practices with regard to accessibility.
    • The best approach for making a Web site accessible is thinking about and building accessibility into it from the early stages in its development.
    • Accessibility should
      • allow use of the Web site by keyboard, mouse, touch screen, screen reader and voice assistant users
      • allow any person regardless of auditory, visual, physical or cognitive abilities to understand and use the Web site
      • prevent the Web site from causing harm, e.g. with motion that can cause eplileptic seziures (Accessibility @MDN)
    • Accessibility benefits everyone, not only people with impairments: it people limited in their choice of device, it can make Web sites faster and more search-engine optimisation (SEO)-friendly

  2. Standards and Laws

    Standards and Laws

    • The Web Content Accessibility Guidelines (WCAG) is a standard published by the World-Wide Web Consortium (w3.org).
    • The WCAG 2 at a Glance site gives a great overview of the standard.
    • Apart from being the right thing to do, accessibility is also gradually being brought into law. In Ireland the Web Accessibility Directive from 2018 will become mandatory for all websites in 2025.

  3. Tools

    Tools

    • HTML, CSS and JavaScript.
    • The accessibility tree is built from the information in the Document Object Model (DOM). It contains semantic information about the Web site that platform-specific Accessibility APIs can transform into representations for assistive technologies.
    • Firefox provides an accessibility tool as part of its Developer Tools, which can be used to inspect a Web site for accessibility via the accessibility tree.
    • The Web Accessibility Initiative - Accessible Rich Internet Applications (WAI-ARIA) W3 Consortium standard contains the specification of constructs for adding semantic information to parts of Web sites that do not contain such information in their HTML content. An introduction to WAI-ARIA can be found on MDN.
