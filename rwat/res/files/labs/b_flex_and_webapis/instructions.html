Flexbox, Web APIs, Conditional CSS

Lab description

The purpose of this lab is to give the student a taste of the wide range of standard material available to the client-side web developer.

Assignment

Develop a client-side application that fulfils the following requirements.

  • The application should implement the sliding puzzle game with 9 tiles (on the Wikipedia page, see the example with a picture of a female superhero, using 9 tiles).
  • The game starts with the user being prompted as to whether they would like to start from scratch or continue their previous game. If this is the first time they are playing, the only option is to start. Then the user is asked about the level of complexity they would like. Once they specify this, the game starts.
  • Internally, the game will start with the arrangement shown in the picture, which is jumbled by random movements during initialisation. The number of random movements that jumble the tiles depends on the level of complexity (you can decide on how many but this can be calibrated when testing).

  • The user should be given the option to save the game at any time during play, so that they can continue to play later.
  • The user can slide a tile into the empty space if they it is in a space adjacent to toe empty space.
  • The game ends when the tiles are arranged in the correct order. When this is achieved, the 9th tile is inserted by the program and a suitable message displayed.
  • The design of the page displaying the completed picture should be slightly different for the print medium, for example including a frame.

Rules:

  • You may use the tiles provided in the images directory or use an image splitter (there are many phone and web apps of this kind) to split an image of your own into 9 parts.
  • Use one HTML file, one CSS file and one Javascript file for the implementation.
  • Use a suitable input element (there are a few that would work) for the user to input the level of complexity.
  • Implement the layout with the use of flexbox.
  • Implement the tile movements using the drag and drop API.
  • Implement the game saving mechanism using the local storage API.
  • Use at least two CSS complex selectors.
  • Use conditional CSS to style the page for printing.

Additional instructions:

  • Clarify the requirements with the tutor in the lab if needed, before starting work on the assignment. With every set of requirements, there is a possibility of under-specification or a lack of clarity. It is the job of the developer to make every effort to fully understand the requirements (as it is, indeed, the job of those laying out the requirements to make the best effort to make them clear and comprehensive - an broader example of the application of Postel's law).
