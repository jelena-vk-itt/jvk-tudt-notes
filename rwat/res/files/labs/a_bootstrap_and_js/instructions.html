HTML, CSS and Javascript Revision with Bootstrap

Lab description

The purpose of this lab is to

  • get back into practice of using your HTML, CSS and Javascript skills
  • get back into practice of following detailed requirements precisely
  • go through a rapid learn-use cycle on a relatively simple technology, the Bootstrap CSS

Very short introduction to Bootstrap

  • Bootstrap provides consistent and responsive styling through a large collection of CSS rulesets and conditionals (see here). It also includes Javascript functionality (we will not be working with this).
  • Styling with Bootstrap is achieved by linking the Bootstrap CSS file into the HTML file and assigning elements in that HTML file to classes defined in Bootstrap. Such an assignment is made as usual, with the class attribute (e.g. <div class="a-bootstrap-class">Content</div>.
  • Documentation on the styles provided by Bootstrap is available, for example, on W3 Schools and on the Bootstrap website.
  • We will be using Bootstrap 5 (you will find the latest version number on the Bootstrap website.

Assignment

  1. Create an HTML file called index.html and a Javascript file called main.js. Prepare the HTML file as follows:
    • add elements !DOCTYPE, html, head, title and body
    • link in the Bootstrap 5 CSS file by using an HTML link element (instructions e.g. on this page, section "Bootstrap CDN")
    • include the Javascript file main.js with a script element that is the last child of the body element
  2. Study the Bootstrap grid system here, then reproduce the layout shown in the picture to the left below, while adhering to the following rules.
    • the whole layout must be contained in a single element belonging to class container-fluid (see here)
    • the layout should be created with the use of Bootstrap row and col-*-* classes, except for the text input element at the bottom
    • the coloured elements must be of type div and be immediate children of the columns
    • all other styling must be achieved with Bootstrap (no other CSS is allowed!)
      • for margins above and below rows see here
      • for built-in Bootstrap colours see here
      • for centering and weighting of text see here
    • the layout should be responsive i.e. 'collapse' into a stack (see the picture to the right below - it shows the upper part of the stack) when the browser window's width is reduced beyond some value; this is automatically achieved by using col-*-* classes, but do test that it actually happens perhaps make modifications to where the margin classes are placed!
           
  3. Add interactivity to the page as follows:
    1. When the first coloured element (with the text 'Here') is clicked, its text should change to the text entered in the input element and its background colour must change (to any colour other than the current one). The changes should happen only if the input element contains a non-empty string. You may edit the HTML file and Javascript file as needed to implement this.
    2. When the second coloured element (with the text 'we') is clicked, the weight of the text visible in all the coloured elements should change: if it is normal it must change to bold and vice versa. You may edit only the Javascript file to implement this.
    3. When the right arrow on the keyboard is pressed, the pieces of text in the coloured elements must be moved so that there is still one piece of text per element but the text appears in alphabetical order going from the top left element to the bottom right element, in the usual reading order. A left arrow keystroke should result in an analogous reverse alphabetical order arrangement, a down arrow keystroke in random order and an up arrow keystroke in the original order (found when the page was first loaded) of the pieces of text. You may edit only the Javascript file to implement this.
