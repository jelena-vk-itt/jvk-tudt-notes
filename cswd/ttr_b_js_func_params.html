Javascript Function Parameters

  1. Parameters vs. Arguments

    Parameters vs. Arguments

    Consider the method console.log(). This method knows how to find the console and print something on it. It doesn't know in advance what needs to be printed, so its definition includes a parameter, stating that the method needs a value to be given to it when it is called. A parameter is essentially a variable, for which the value is provided by the caller of the function or method. The rules for naming parameters are the same as for any Javascript identifier. They are listed here.

    The value provided for the parameter when the function is called, is named an argument. Another way to to say that the value is provided is 'the argument is passed in' to the function or method. In the case of console.log, once the method receives the argument, it knows what to print to the console: it prints the argument (which can be a string, a number or anything else).

    Parameters are specified in the parameter list (in brackets after the function name) and arguments are specified in the argument list (also brackets after the function name). Despite looking the same, the two lists are quite different: one is a list of variables, the other a list of values for those variables.

    The alert method pops up a dialogue window with some text in it. It also does not know in advance what the message should be and its definition includes a parameter i.e. specifies that an argument can be passed into the method when it is called. The argument is displayed by the method in the dialogue window.

    Try to call alert() from the console without arguments. What happens? There is no error but the dialogue window has no content when it pops up. The parameter is optional (documentation).

    Now call console.log() passing it several arguments of different types, for example console.log("hello", 3, true, 4.5). All the values are printed to the console as console.log() can take any number of arguments (documentation).

  2. Why use parameters?

    Why use parameters?

    • For the purpose of reusing functionality with different data (the parameter(s) contain the data)
    • For the purpose of reusing functionality in slightly different contexts (the parameter(s) define the context)
    • Because functions and methods most often work with data that is unknown at the time when the function is being defined
      • The parameter is a placeholder for the unknown data
      • Several parameters represent several pieces of unknown data
