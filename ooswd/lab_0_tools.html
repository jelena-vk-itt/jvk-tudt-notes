  2. Running Python code

    Running Python code

    Note: Sample commands are preceded by the characters '> ', representing a cursor. If the command is shown as 

    > python
    then all you have to type in at the command line is python.

    Python code can be invoked in various ways. In this module, we will be using the following methods.

    • Interactively from the command line (command-by-command)
      1. open a command line window on your computer (this will differ from OS to OS)
      2. type in
        
> python
      3. check that the cursor >>> is being displayed; this means that you are in the Python environment and can type Python code into the command line
      4. type in 
        
> 3 + 4
        and press Enter; what happens then?
      5. when running the Python interpreter interactively like this, expressions are evaluated according to the rules of the language and the result of the evaluation displayed on the console
      6. we will be trying out a few more commands in class...
    • As a script from the command line
      1. create and save a text file called lab1.py with the following line in it:
        3 + 4
      2. on the command line, change the current directory to the one containing the file lab.py
      3. type in
        python lab1.py
        what happens then?
      4. in batch mode, the interpreter does not display the results of evaluating expressions, so, while 3 + 4 is added, nothing further is done with the result; we must explicitly specify that we want the result printed and to do this you should change the line in lab.py to 
        print(3 + 4)
      5. run the script again as previously; this time you should see 7 printed out in the console

  4. Notes on the notes

    Notes on the notes

    • There are code snippets (examples) and exercises, such as the sample ones below, throughout the module notes.

      Treat the code snippets as starting points for exploration of a concept or topic, devising and trying out further examples yourself.

      The exercises are specific and should be carried out according to the instructions. Apart from being Python-learning exercises, they double as practice in requirement gathering and interpretation.

      For example, write a short Python program that reads in two numbers from the console, adds them up and prints the result.

    • Links shown in bold maroon point to reference material. For example, here are all the logical operators.
    • Angular brackets containing descriptive text (e.g. <your name>) are placeholders, used to describe required input or other literals. For example,
      • if an expected input string value is described as <name> then the actual value could be John or Mary
      • if a value is described as <date of birth in format dd/mm/yy> then the actual value might be 01/01/01

      The actual value should not include angular brackets.

      • text printed by the programe is displayed in white
