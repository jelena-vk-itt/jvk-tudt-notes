Notation in assignment specifications
  • Angular brackets containing descriptive text are placeholders, used to describe required input or other literals. For example,
    • if an expected input string value is described as <name> then the actual value could be John or Mary
    • if a value is described as <date of birth in format dd/mm/yy> then the actual value might be 01/01/01

    The actual value should not include angular brackets.

  • If a command line window is shown to demonstrate requirements,
    • text printed by the programe is displayed in white
    • user input is displayed in red
    For example,
    Please enter an number: 42

Exercise 1: Processing a number list

  • Write a function that takes a list of numbers and prints the numbers separated by commas.
  • Define the list [1, 2, 3, 5, 7] and print it using your function.
  • For each of the following cases, use a list comprehension with a lambda expression to map the list to a new list, then print it out
    1. the new list contains the squares of the values in the original list
    2. the new list contains the square roots, rounded to two decimal places, of the values in the original list
    3. the new list contains only the odd numbers from the original list

Exercise 2: Processing a list of strings

  • Define a list of names
  • Write a function that takes a string and returns true if the string contains less than two vowels
  • Use the function in a list comprehension that converts strings in the original list to all-lowercase and fiters it by the function you wrote in the previous step.
  • Print the original list and the list constructed by the list comprehension.
