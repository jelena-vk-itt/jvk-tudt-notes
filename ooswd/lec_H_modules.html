    Importing modules

    We have already seen that some modules that are installed with Python have to be imported in order to be used, because they are not loaded into the interpreter by default. Examples are the random module, which we have used before, and the datetime module.

    There are several variations of the import syntax, as seen in the next code snippet

    Developers can use modules to organise their own code, since large files, e.g. containing multiple classes, can be hard to read and maintain.

    With imports, code can be placed in multiple files and the dependencies between them managed with the import statement.
