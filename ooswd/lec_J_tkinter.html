    Widgets

    A GUI is built from various elements, called widgets. With tkinter, these are always added to a parent object, which can be either the root object or another widget.

    Each widget has many configuration options, a lot of which will not be used. These can be found in widget documentation (see links at bottom of article). However, some options are essential to the widget's usefulness, for example, the configuration option text of a Label widget, which is used to specify the label's text content. In the example above we see this option being set via a parameter called text.

    Apart from being created, a widget must be placed in its parent with one of three available mechanisms (pack, place, grid). The pack system is older and harder to use than grid, while place gives full control but is also harder to use. Thus, we use grid in all the examples. In the code above, the label is placed in the top left corner of the GUI, as expected with the usual meaning of row and column indices.

    Configuration options

    We have seen configuration options passed in as arguments to the widget object constructor. They can also be set as attributes on the widget object or as arguments to the configure method.

    In the example above, the configuration option anchor specifies the position of the text in the label ('w' for west).

    Variables

    Some of the configuration options take variables rather than values. This allows for the value of these options to change dynamically.

    In the example above, once the variable txtVar is associated with the widget label, any changes in the variable's value will be reflected in the label text. After the code in the example runs, the text in the label widget is 'Goodbye!', even though the value of the variable was 'Hello!' when the variable was associated with the widget.
