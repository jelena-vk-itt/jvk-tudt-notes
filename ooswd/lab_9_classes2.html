Another Classes Lab

Starting with the solution to the previous lab (yours or the published one), complete the following tasks.

  1. Modify the Shop class to make methods static if they do not access the object members (there are three such methods).
  2. Modify the Product class so that it uses properties for name and stock.
  3. Add the method get_department() to class Product. It should return the string "General".

    Then define two subclasses of Product:

    • FoodProduct, which should include an additional property, expiryDate, and override the method get_department(), returning a suitable constant string
    • ClothingProduct, which should include an additional property, season, and override the method get_department(), returning a suitable constant string. The setter for season should report an error and not modify the object if the parameter's value is not one of spring, summer, autumn or winter.
  4. Define the __str__ method on classes Product, FoodProduct and ClothingProduct.
  5. Define detailed tests for all the new functionality in the main.py file and run them.
  6. Also in the main.py file, define a list of four objects, two of each subclass. Use a loop to print out all four objects one after the other.
  7. Additional challenge: Define the comparison dunder methods in the class Product, in order to facilitate alphabetical ordering of products by name. Test the solution by ordering and printing the objects created in the last step.
