Exercise 1

Starting with your solution (or the given one) to exercise 3 in lab assignment Revision, strings, lists (week 2):

  1. improve the code by refactoring it to use constructs learnt since (e.g. loops) and, specifically, implement a function that orders a list given preference numbers like those expected in the program
  2. handle any number of items greater than 1, rather than 5
  3. handle the following errors, with exceptions or otherwise, as appropriate:
    • the number of items entered is less than 2
    • the count of the preference ordering numbers entered does not match the item count
    • any of the preference ordering numbers are not integers in the correct range (e.g. 1 - 5)

Exercise 2

Starting with your solution (or the given one) to lab assignment Classes (week 8), modify it so that the following errors are reported as exceptions:

  1. the argument passed to the Product name setter is not a non-empty string
  2. the argument passed to the Product stock setter is not a non-negative integer
  3. the argument passed to the Shop constructor is not a non-empty string
  4. an already-listed product name is passed to the Shop method add_product
  5. a non-listed product name is passed to any of the Shop methods remove_product, restock and sell
  6. an invalid stock increase value is passed to Shop method restock
  7. an invalid stock decreease value is passed to Shop method sell

Write some code in main.py to test each instance of a raised exception by using a try-except statement.
