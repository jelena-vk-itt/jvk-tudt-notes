Notation in assignment specifications
  • Angular brackets containing descriptive text are placeholders, used to describe required input or other literals. For example,
    • if an expected input string value is described as <name> then the actual value could be John or Mary
    • if a value is described as <date of birth in format dd/mm/yy> then the actual value might be 01/01/01

    The actual value should not include angular brackets.

  • If a command line window is shown to demonstrate requirements,
    • text printed by the programe is displayed in white
    • user input is displayed in red
    For example,
    Please enter an number: 42

Exercise 2: Calculator

  • Write a function that takes three parameters, expecting two number arguments and a string with one of the following values: '+', '-', '*' or '/'. The function should perform the operation corresponding to the given string on the two number parameters and return the result of the operation. It should handle all cases of invalid arguments by printing an informative message and returning None. Name your function and parameters suitably, based on their meaning.
  • Use the function in a program where the user is asked to enter two numbers and an operation and where the result of the operation is displayed to the user. The function will handle any invalid inputs, but you will have to check if the value returned by the function is None, in order to decide if you should print a result. If result is the value returned by a call to the function, you can use either of the following conditions: 
    
if not result is None:
    # do something
 
if result:
    # do something
  • List a comprehensive tests for the program.
  • Perform the tests and fix any problems with the program that are revealed by the testing. Repeat this process until the program works correctly.
